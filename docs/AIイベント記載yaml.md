## AIイベントプログラム作成で、何故yamlを利用する必要があるのかを説明

　通常 claudeCode などでは、普通に「チャット的な形」で「日本語でプログラム作成の対応を指示」するが、この場合以下の問題・懸念点が起きる。

1. 日本語で指示出しした内容が、その通りAIが意図を解釈して、忠実に対応できるか否かは「指示して実行」したソースコードを確認する必要がある。
2. 一方で「日本語で指示した内容」が「毎回全く同じソースコード結果」とはならない可能性。
3. あと、AIと同じく「人間でも同じく日本語の文書の解釈が、人によって違う」わけで、要するに「引き継ぎしたり」の場合の「ズレ」が出てきてしまう問題。
4. 引き継ぎなどにおいて、フリーハンドの「日本語の文書で指示をAIに出す」事で、様々な「負債が溜まっていく」事になる。

　これらは「実際のプログラム開発」においても「運用フェイズに移行後に、同じような事が起きる」が、これらが「フリーハンドの日本語」である場合は、恐らく「この負債が溜まっていく頻度・割合」ってのは「フリーハンド」であることから増えていく事が予想される。

　そのため、ある程度「テンプレートに則った形」で「日本語でプログラムじゃない」けど、ある程度「書き方の共通化」ができるものとして、提案するのが `yaml` での記載であり、これら「脱フリーハンド化」の対策として、vja では「AIに指示を出す＝yaml」と言う形で行うようにする。

　これによって先程の「負債が溜まる問題」は「ある程度回避できる」ようになる。

## AIイベント記載yaml例

```yaml
利用テーブル:
    - users # ユーザー情報

目的: 画面の入力内容を「ユーザー（users）」データとして保存し、次の画面へ進む

もし以下のとき:
    - 入力されていない項目がある場合:
          アクション: 「入力されていない項目があります」異常系終了

データ操作:
    - 保存先テーブル: users
    - ルール:
          - 入力内容に同じ「name」が存在する場合: データを上書き（更新）する
          - 存在しない場合: 新しく追加する

結果:
    - 正常系:
          画面遷移: 画面2
    - 異常系: エラー内容をポップアップ表示して、処理を中断
```

### 説明

- 利用テーブル:
  [必須]対象イベントでテーブル利用をする場合は記載してください。

- 目的:
  [必須]AIにこの処理のヒントを与える上で重要なので、概要的な内容を記載して下さい。

- もし以下のとき:
  [任意]入力バリデーション的な必要な入力項目が入ってない時など、一番最初に記載しておくと、バリデーションチェックしてくれます
    - 入力されていない項目がある場合:
      入力されていない項目をチェックしてもらう＝validationをざっくり指示する場合の条件として設定
        - アクション:
          実際の「処理」として行う内容を記載する

- データ操作:
  [任意]アプリがsqlite3(bun.js付属の)にデータ検索・書き込みを行う場合に記載する
    - 保存先テーブル:
      保存先のテーブル名を設定します。
    - ルール:
      処理条件（今回の場合は upsertを指示する）ものとして定義.
        - 入力内容に同じ「name」が存在する場合
          select で name が存在した場合は、updateする.
          またここでの name は 対象フォームの入力枠の名前: name が対象となる.
        - 存在しない場合
          insertする

- 結果:
  [必須]この定義が無い場合は「正常終了しても、何もしないで、当該フォーム画面の表示に戻る」
    - 正常系:
      正常処理の場合の処理を定義する
        - 画面遷移:
          画面遷移を行う場合は、このように定義する
    - 異常系:
      主にExceptionが発生した場合にどうするかを定義する。
      ただ、Exception系が発生した場合は、そもそもエラーアラートと、ログ出力されるので、エラーが起きた場合にどう振る舞うかを定義する

### 考え方

　プログラムには「if文やfor文」があり、そして「変数」があるが、これらが「プログラムを構成する中核機能」であると同時に「プログラムを学ぶ・プログラムを読む」に対する「障壁」であると言える。

　一方で「多くの場合、プログラム」はやりたいこととして「if文：分岐=もしも」を積み重ねて、処理を実装していくわけで、今回の yaml の内容は、これら「大まかな処理の分岐」を記載して、AIにプログラムを組んでもらう形とする。

　これら分岐を表すものとして、yamlが一番テンプレート的な記載として「理にかなってる」し、人が実装しやすいフォーマットであるわけで、それが上の「AIイベント記載yaml例」であり、これによって「ソースコードを作成してもらうAIに、わかりやすく伝わる」ので、AIが間違ったソースコードを組みにくい結果となる。

　あと「人が見ても内容が理解しやすい」正に「第２のCOBOL言語」的な形として vja開発環境の使い方を理解している人なら、非エンジニアでも「変更対応」ができるものとして「yamlフォーマット」が非常に強力である。

### その他補足やアイデア的な内容メモや補足

- 現在アイデアベースだが、vja のイベント yaml入力の所に「利用テーブル選択」ができる形にしておくと、この利用テーブルをユーザに入力させなくてよくなりそう。
- AIには、利用テーブル一覧に対する、テーブルカラムや not null などのスキーマー情報を渡すようにする。
- 対象フォーム内の入力情報名は 入力項目名一覧を、AIに渡す形にする。
