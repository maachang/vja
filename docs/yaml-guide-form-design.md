# 🎨 VJA 画面デザイン自動生成 YAML ガイド

## ～ YAMLを書くだけで、フォームの画面イメージができる ～

---

## はじめに

VJAでは、ウィジェットを1つずつ手作業で配置する前に、**「どんな画面にしたいか」を YAML 風の依頼文で書く**ことで、ローカル AI にウィジェットの配置（画面イメージ）を自動生成させることができます。

これは「[イベント処理用のYAML](yaml-guide-startup.md)」（ボタンを押したときの処理などをAIにJavaScriptとして生成させるもの）とは**別の機能・別のYAML**です。

- イベント処理用YAML → 「クリックしたら何をするか」をAIがコード化する
- 画面デザイン用YAML（本ガイド） → 「どんなウィジェットをどこに置くか」をAIが配置する

**画面デザイン用YAMLで画面の土台（ウィジェット配置）を作ってから、各ウィジェットのイベントに対してイベント処理用YAMLを書く**、という順番で使うのがおすすめです。

---

## 起動方法

1. フォームを選択した状態（ウィジェットは何も選択しない）にする
2. プロパティパネルの **「🤖 AIでフォーム設計…」** ボタンをクリック
3. エディタが開くので、YAML形式で依頼内容を記述する
4. 必要であれば「追加指示」欄に補足を入力
5. **「🤖 生成」** ボタンをクリック

> 事前に [AI 接続設定](user-guide.md#14-ai-接続設定) でエンドポイントを設定しておく必要があります。

---

## YAML の基本構造

```yaml
# フォームデザイン定義.

説明: ユーザーの新規登録を行う画面
フォームレイアウト: ラベルと入力欄は横並び（同じY座標）にし、ボタンは右下に配置する。
参照テーブル:
  - users
入力項目:
  - ユーザー名
  - メールアドレス
  - パスワード
アクション項目:
  - 登録
```

### 各キーの意味

| キー | 必須 | 説明 |
|------|------|------|
| `説明:` | 任意 | 画面の目的を一言で。空欄の場合はフォームのプロパティに設定済みの「説明」が自動的に使われる |
| `フォームレイアウト:` | 任意 | 配置の方針・デザインの指示。指定した場合、**最優先の制約**としてAIが解釈する |
| `参照テーブル:` | 任意 | 入力項目の元になるテーブル名（list定義）。指定すると、そのテーブルに存在しないカラム名を勝手に作られるのを防げる |
| `入力項目:` | 推奨 | 配置したい入力欄の一覧 |
| `アクション項目:` | 推奨 | 配置したいボタンの一覧。**指定した数だけボタンが生成される**（AIが勝手に増減しない） |

> `#` で始まる行はコメントです。

---

## 「フォームレイアウト」の書き方

`フォームレイアウト:` はAIが最優先で解釈するため、画面イメージを一番左右するキーです。具体的に書くほど狙った配置に近づきます。

### 書き方の例

```yaml
フォームレイアウト: ラベルと入力欄は横並び（同じY座標）にし、ボタンは右下に配置する。
```

```yaml
フォームレイアウト: 2カラム構成にして、左側に入力項目、右側に一覧表示用のスペースを空けておく。
```

```yaml
フォームレイアウト: ラベルは入力欄の上に配置し、上から下へ縦一列に並べる。ボタンは画面下部中央に配置する。
```

### コツ

- 「横並び」「縦一列」「2カラム」など、**配置の方向**を明示する
- ボタンの位置（右下・中央・上部など）を明示する
- 指定がない場合、AIは `入力項目:` に書かれた順番で上から下へ配置する

---

## 「参照テーブル」の使い方

入力項目がDBのカラムに対応する場合、`参照テーブル:` にテーブル名を指定すると、AIがそのテーブル定義を参考にします。

```yaml
説明: ユーザー情報の編集画面
参照テーブル:
  - users
入力項目:
  - ユーザー名
  - メールアドレス
  - 生年月日
アクション項目:
  - 更新
```

> 参照テーブルに存在しないカラム名を、AIが勝手に入力項目として追加することはありません。テーブルにない項目を使いたい場合は `入力項目:` に明示してください。

---

## 選択肢（ドロップダウン・リスト選択）の指定について

`selectBox`（ドロップダウン選択）や `listBox`（リスト選択）に相当する入力項目を書くと、AIが依頼文の内容から具体的な選択肢を推測して生成します。選択肢を具体的に指定したい場合は、`入力項目:` に選択肢を書き添えてください。

```yaml
説明: 問い合わせ管理画面
入力項目:
  - ステータス（未処理/処理中/完了）
  - 担当者
アクション項目:
  - 保存
```

- 選択肢を明示すると、AIはその通りの選択肢で生成します
- 選択肢を明示しない場合、AIが依頼内容や参照テーブルの内容から一般的な選択肢を推測して生成します（空の選択肢になることはありません）
- 生成後、選択肢の内容を変更したい場合は、配置後のウィジェットのプロパティパネル（「✏ 項目編集…」）から編集できます

### 表示名と内部値（Value）を分けたい場合

「画面に表示する文言」と「実際にコードやSQLで使う値（カラム値など）」を分けたい場合は、`表示名: 内部値` の形式で列挙してください。

```yaml
説明: horse_info の検索画面
参照テーブル:
  - horse_info
入力項目:
  - 検索ワード
  - 検索条件選択項目: selectBox で key=表示名, value=Value
    - 馬名: name
    - 父馬: father
    - 母馬: mother
    - 性別: sex
アクション項目:
  - 検索
```

- `馬名: name` のように「表示名: 内部値」を明記すると、AIは表示名と内部値を分けて選択肢を生成します（`name`カラムの検索に使う、といった実装につながる）
- 生成後の選択肢は、プロパティパネルの「✏ 項目編集…」で表示名／Value（省略可）の一覧として確認・修正できます

---

## テーブル（datagrid）の指定について

一覧表示用の `テーブル`（datagrid）ウィジェットも、画面デザインの中で生成できます。「一覧」「テーブル」「表形式」などのキーワードや、参照テーブルの指定と合わせて `アクション項目:` や `フォームレイアウト:` に書くことで、AIがカラム構成を生成します。

```yaml
説明: ユーザー一覧を検索・表示する画面
フォームレイアウト: 上部に検索欄と検索ボタンを横並びで配置し、その下に一覧表示用のテーブルを配置する。
参照テーブル:
  - users
入力項目:
  - 検索文字列
アクション項目:
  - 検索
```

- 参照テーブルを指定すると、AIはそのテーブルのカラム定義に基づいてテーブルのカラム構成を生成します
- 参照テーブルを指定しない場合、AIは依頼内容から一般的なカラム構成を推測して生成します
- 生成後、カラム構成を変更したい場合は、配置後のウィジェットのプロパティパネル（「✏ カラム編集…」）から編集できます

### カラム名（データ用）と表示名を分けたい場合

datagridの各カラムには「実データのカラム名」と「画面に表示する見出し」の2つがあります。参照テーブルを指定した上で、「テーブル項目を表示して、カラム名、表示名を設定する」のように明記すると、AIが実際のカラム名と日本語の表示名の両方を設定した状態で生成します。

```yaml
説明: horse_info の検索画面
参照テーブル:
  - horse_info
入力項目:
  - 検索結果表示枠: datagrid
    - horse_info: テーブル項目を表示して、カラム名、表示名を設定する
アクション項目:
  - 検索
```

- 参照テーブルのカラム名（例: `name`, `father`）がデータ用のカラム名として使われ、日本語の見出し（例: `馬名`, `父馬`）が表示名として設定されます
- 生成後の内容は、プロパティパネルの「✏ カラム編集…」でカラム名／表示名／幅の一覧として確認・修正できます

## 記述例

### 1. シンプルなログイン画面

```yaml
説明: ログイン画面
フォームレイアウト: ラベルと入力欄は横並び（同Y座標）にし、ボタンは右下に配置する。
入力項目:
  - ID
  - パスワード
アクション項目:
  - ログイン
```

### 2. 一覧検索画面（テーブル参照あり）

```yaml
説明: users テーブルの一覧を検索・表示する画面
フォームレイアウト: 上部に検索欄と検索ボタンを横並びで配置し、その下に一覧表示用の広いスペースを確保する。
参照テーブル:
  - users
入力項目:
  - 検索文字列
アクション項目:
  - 検索
```

> この記述により、検索欄・検索ボタンに加えて、`users` テーブルのカラム定義に基づいたテーブル（datagrid）も生成されます。生成後、実際の検索処理は[イベント処理用YAML](yaml-guide-startup.md)の「検索してテーブルビューに表示する」の要領で書いてください。

### 3. 新規登録フォーム（2カラム構成）

```yaml
説明: 商品マスタの新規登録画面
フォームレイアウト: 2カラム構成にする。左カラムに基本情報、右カラムに価格・在庫情報を配置し、ボタンは画面下部中央に置く。
参照テーブル:
  - products
入力項目:
  - 商品名
  - 商品コード
  - 価格
  - 在庫数
アクション項目:
  - 登録
  - キャンセル
```

## 生成後の挙動（重要）

- **現在のフォームに配置済みのウィジェットは一旦全て削除され、AIが生成した内容に置き換わります。**
- 削除されるのはウィジェットの配置だけでなく、**それらに設定済みのイベント処理（コード）も含みます**。
- 実行前に確認ダイアログが表示されるので、既存の画面を編集中の場合は内容をよく確認してください。
- 誤って実行してしまった場合も、`Ctrl+Z` で元に戻せます。
- `selectBox` / `listBox` の選択肢一覧、`datagrid` のカラム構成もAIが生成します（依頼文で明示しない場合は、AIが内容から推測して生成します）

## よくある書き方のコツ

### ✅ 良い書き方

```yaml
説明: ユーザーの新規登録画面
フォームレイアウト: ラベルは入力欄の上に配置し、上から下へ縦一列に並べる。ボタンは右下に配置する。
参照テーブル:
  - users
入力項目:
  - ユーザー名
  - メールアドレス
アクション項目:
  - 登録
```

- `フォームレイアウト:` で配置方針を具体的に書く
- `入力項目:` は実際に必要なものだけを過不足なく書く
- `参照テーブル:` を指定して、カラム名のブレを防ぐ

### ❌ 避けた方が良い書き方

```yaml
説明: いい感じの画面
入力項目:
  - いろいろ
アクション項目:
  - 実行
```

- 「いい感じ」「いろいろ」のような曖昧な表現ではAIが意図を読み取れません
- 入力項目・アクション項目は具体名で書きましょう

## 実際の流れ（まとめ）

```
① プロパティパネルから「🤖 AIでフォーム設計…」を開く
② 説明・フォームレイアウト・参照テーブル・入力項目・アクション項目をYAMLで書く
③ 「🤖 生成」ボタンをクリック（既存ウィジェットは確認の上で削除される）
④ 生成された画面イメージを確認し、必要に応じて手動で微調整する
⑤ 各ウィジェットのイベントに対して、イベント処理用YAML（yaml-guide-startup.md / yaml-guide-engineer.md）を書いていく
```

## YAML キー一覧（早見表）

| キー | 用途 | 例 |
|------|------|----|
| `説明:` | 画面の目的を記述 | `説明: ユーザーの新規登録画面` |
| `フォームレイアウト:` | 配置方針・デザイン指示（最優先） | `フォームレイアウト: ボタンは右下に配置する` |
| `参照テーブル:` | 入力項目の元になるテーブル名 | `- users` |
| `入力項目:` | 配置したい入力欄の一覧 | `- ユーザー名` |
| `アクション項目:` | 配置したいボタンの一覧（数は厳守される） | `- 登録` |
