# 📝 VJA YAML プログラム命令ガイド(スタートアップマニュアル)
## ～ VBA/Excel 経験者のための入門マニュアル ～

---

## はじめに

VJAでは、イベント（ボタンクリックなど）に対する処理を **YAML** という形式で記述します。
AIがそのYAMLを読んで、実際に動くJavaScriptコードを生成します。

「プログラムを書く」というより「仕様書を書く」イメージです。
VBAで言えば、コードを書く前に「こういう処理をしたい」とメモを書く感覚に近いです。

---

## YAML の基本構造

すべてのイベントYAMLは以下の構造になっています。

```yaml
# イベント: Click (btnSave)
説明: 入力内容を保存する
利用テーブル:
  - test_table
検証: 入力確認用
アクション:
  - やりたいことを日本語で書く

正常終了: なし
```

### 各キーの意味

| キー | 必須 | 説明 |
|------|------|------|
| `# イベント:` | 自動 | どのイベントか（自動入力） |
| `説明:` | 推奨 | この処理の目的を一言で |
| `利用テーブル:` | 任意 | DBテーブルを使う場合に指定(list定義) |
| `検証:` | 任意 | バリデーション定義を使う場合 |
| `アクション:` | 必須 | 実際の処理内容（複数書ける） |
| `正常終了:` | 推奨 | 処理成功後の動作 |
| `エラー終了:` | 推奨 | エラー時の動作 |

> `#` で始まる行はコメントです。不要な場合はそのまま残してOKです。

---

## アクションの書き方

`アクション:` の下に `- ` （ハイフン＋スペース）を付けて処理を日本語で書きます。
VBAで言えば、コメントに処理の流れを書くイメージです。

```yaml
アクション:
  - txtName の入力値を取得する
  - 入力値が空なら「名前を入力してください」とダイアログを表示して処理を終了する
  - users テーブルに名前を保存する
```

### 条件分岐（IF文）

インデントを使って条件を書きます。

```yaml
アクション:
  - 確認ダイアログ「保存しますか？」を表示する:
      - YES の場合: users テーブルに保存して「保存しました」とトースト表示する
      - NO の場合: 何もしない
```

VBAで書くと以下に相当します：
```vba
If MsgBox("保存しますか？", vbYesNo) = vbYes Then
    ' users テーブルに保存
    MsgBox "保存しました"
End If
```

### 繰り返し処理

```yaml
アクション:
  - users テーブルの全レコードに対して以下を繰り返す:
      - 各レコードの name を listBox1 に追加する
```

## よく使うアクションのサンプル集

### 1. ダイアログ表示

```yaml
アクション:
  - 「処理が完了しました」とダイアログを表示する
```

```yaml
アクション:
  - 「削除しますか？」と YES/NO の確認ダイアログを表示する:
      - YES の場合: 削除処理を行う
      - NO の場合: 何もしない
```

---

### 2. ウィジェット（画面部品）の操作

```yaml
アクション:
  - txtName の入力値を取得して name 変数に入れる
  - lblResult に「こんにちは、{name}さん」と表示する
  - btnDelete を非表示にする
  - txtMemo を空にする
```

VBAの `TextBox1.Value` / `Label1.Caption` に相当します。

---

### 3. テーブル（DB）からデータを取得する

`#利用テーブル:` を有効にして、使うテーブル名を指定します。

```yaml
説明: ユーザー一覧を表示する
利用テーブル:
  - users

アクション:
  - users テーブルから全件取得して listBox1 に名前一覧を表示する

正常終了: なし
エラー終了: ログとトーストにエラーを出力する
```

---

### 4. テーブルにデータを保存する

```yaml
説明: 入力内容をDBに登録する
利用テーブル:
  - users

アクション:
  - txtName と txtEmail の入力値を取得する
  - users テーブルに name と email を INSERT する

正常終了: 「登録しました」とトースト表示する
エラー終了: ログとトーストにエラーを出力する
```

---

### 5. 検索してテーブルビューに表示する

```yaml
説明: 検索条件でユーザーを絞り込む
利用テーブル:
  - users

アクション:
  - ローディングを表示する
  - txtSearch の入力値を取得する
  - 入力値が空でない場合: users テーブルから name が部分一致するレコードを検索して tableView1 に表示する
  - 入力値が空の場合: users テーブルから全件取得して tableView1 に表示する

正常終了: ローディングを非表示にする
エラー終了: ログとトーストにエラーを出力する
```

---

### 6. 画面遷移（フォーム移動）

VBAでいう「別のフォームを開く」に相当します。

```yaml
アクション:
  - 画面「Form2」に遷移する
```

値を渡して遷移する場合：

```yaml
アクション:
  - tableView1 で選択された行の id を取得する
  - パラメータ「userId」に id をセットして画面「Form2」に遷移する
```

遷移先（Form2）でパラメータを受け取る：

```yaml
アクション:
  - パラメータ「userId」を取得して txtId に表示する
```

---

### 7. バリデーション（入力チェック）

GUIで定義したバリデーションをYAMLから呼び出せます。

```yaml
説明: 入力チェックを行ってから保存する
検証: 登録入力チェック
アクション:
  - users テーブルに入力値を INSERT する

正常終了: 「登録しました」とトースト表示する
エラー終了: ログとトーストにエラーを出力する
```

`検証:` に定義名を書くと、AIが生成するコードの先頭に自動的にバリデーション処理が挿入されます。バリデーション定義はツールバーの「✅ 検証」から管理できます。

---

### 8. ローディング表示

時間のかかる処理（DB検索など）の前後に表示します。

```yaml
アクション:
  - ローディング「データを読み込み中...」を表示する
  - （時間のかかる処理）
  - ローディングを非表示にする
```

> エラーが起きてもローディングが消えるよう、AIが `try/finally` パターンで生成します。

---

### 9. セッション（ログイン情報の保持）

画面をまたいで値を保持したい場合に使います。
VBAでいうグローバル変数に相当します。

```yaml
アクション:
  - セッション「loginUser」にユーザー情報を保存する
```

```yaml
アクション:
  - セッション「loginUser」を取得して lblUser に表示する
```

## 正常終了・エラー終了の書き方

### 正常終了

```yaml
正常終了: なし
```

```yaml
正常終了: 「保存しました」とトースト表示する
```

```yaml
正常終了: ログに「処理完了」と出力する
```

### エラー終了

```yaml
エラー終了: ログとトーストにエラーを出力する
```

```yaml
エラー終了: 「エラーが発生しました」とダイアログを表示する
```

> `エラー終了:` を書いておくと、AIが `try/catch` を自動で生成します。

## よくある書き方のコツ

### ✅ 良い書き方

```yaml
アクション:
  - txtName の入力値が空の場合、「名前を入力してください」とダイアログを表示して処理を終了する
  - users テーブルに name カラムとして INSERT する
```

- **具体的なウィジェット名・テーブル名・カラム名**を明示する
- 条件や順序を **箇条書き** で分けて書く
- 「〜の場合」「〜したら」など **日本語で自然に書く**

### ❌ 避けた方が良い書き方

```yaml
アクション:
  - いろいろ処理する
  - データを保存する
```

- 曖昧すぎるとAIが意図を読み取れません
- テーブル名・ウィジェット名は正確に書きましょう

## 実際の流れ（まとめ）

```
① フォームデザイナーでボタンなどを配置する
② イベントを開いてYAMLを書く
③ 「AIで生成」ボタンをクリック
④ 生成されたJavaScriptを確認して実行する
⑤ うまく動かなければ、YAMLを修正してAIに再生成を頼む
```

VBAと同じように「書いて → 動かして → 直す」を繰り返すことで、
自然とプログラムの読み方も身についてきます。

## YAML キー一覧（早見表）

| キー | 用途 | 例 |
|------|------|----|
| `説明:` | 処理の目的を記述 | `説明: ログイン処理` |
| `利用テーブル:` | 使用するDBテーブル | `- users` |
| `検証:` | バリデーション定義名 | `検証: ログイン入力チェック` |
| `アクション:` | 処理の内容 | `- txtName の値を取得する` |
| `正常終了:` | 成功後の処理 | `正常終了: トーストで完了を表示` |
| `エラー終了:` | エラー時の処理 | `エラー終了: ログとトーストに出力` |
